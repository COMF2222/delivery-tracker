# Delivery Tracker

Backend-сервис для отслеживания посылок, написанный на Go.

## Реализованный функционал

### Посылки

* Создание посылки
* Генерация уникального трек-номера
* Получение информации о посылке по трек-номеру
* Получение списка посылок
* Пагинация списка посылок
* Фильтрация посылок по статусу
* Архивация доставленных посылок

### Статусы посылок

Поддерживаются следующие статусы:

* CREATED
* PURCHASED
* WAREHOUSE
* IN_TRANSIT
* CUSTOMS
* ARRIVED
* DELIVERED

Реализована проверка переходов между статусами. Пропуск статусов и откат назад запрещены.

Пример цепочки:

CREATED → PURCHASED → WAREHOUSE → IN_TRANSIT → CUSTOMS → ARRIVED → DELIVERED

### История изменений

При каждом изменении статуса автоматически создаётся запись в истории статусов посылки.

### Фотографии посылок

Реализована загрузка фотографий посылок через `multipart/form-data`.

Поддерживается:

* загрузка файла
* сохранение файла на диск
* сохранение пути к файлу в базе данных
* ограничение размера файла
* проверка расширения файла

### Пользователи

Реализована работа с пользователями:

* создание пользователя
* получение информации о текущем пользователе
* деактивация пользователя

Поддерживаются роли:

* admin
* manager

### Аутентификация и авторизация

Реализовано:

* хеширование паролей через bcrypt
* JWT-аутентификация
* middleware для проверки JWT
* ролевая модель доступа RBAC
* передача данных пользователя через request context

### Аудит действий

Фиксируются действия пользователей:

* изменение статуса посылки
* архивация посылки
* деактивация пользователя

Для каждого действия сохраняются:

* пользователь, выполнивший действие
* тип действия
* старое значение
* новое значение
* сущность, над которой выполнено действие

### Транзакции

Следующие операции выполняются в рамках одной транзакции:

* изменение статуса + история + аудит
* архивация посылки + аудит
* деактивация пользователя + аудит

### Swagger

Реализована Swagger-документация API.

Swagger UI доступен по адресу:

```text
/swagger/index.html
```

## API

### Авторизация

```http
POST /api/v1/auth/login
```

### Пользователи

```http
POST /api/v1/users
GET /api/v1/users/me
PATCH /api/v1/users/deactivate?id={id}
```

### Посылки

```http
GET /api/v1/parcels?page={page}&limit={limit}&status={status}
POST /api/v1/parcels
GET /api/v1/parcels/track?track_number={track_number}
PATCH /api/v1/parcels/status?id={id}
POST /api/v1/parcels/photos?id={id}
PATCH /api/v1/parcels/archive?id={id}
```
