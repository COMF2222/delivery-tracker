# Delivery Tracker

Сервис для отслеживания посылок, написанный на Go.

## Реализованный функционал

### Посылки

* Создание посылки
* Генерация уникального трек-номера
* Получение информации о посылке по трек-номеру
* Архивация доставленных посылок

### Статусы посылок

Поддерживаются следующие статусы:

* CREATED
* PURCHASED
* WAREHOUSE
* IN_TRANSIT
* CUSTOMS
* ARRIVED
* DELIVERED

Реализована проверка переходов между статусами. Пропуск статусов запрещён.

Пример цепочки:

CREATED → PURCHASED → WAREHOUSE → IN_TRANSIT → CUSTOMS → ARRIVED → DELIVERED

### История изменений

При каждом изменении статуса автоматически создаётся запись в истории.

### Фотографии посылок

Поддерживается добавление фотографий к посылке.

### Пользователи

Реализована работа с пользователями:

* создание пользователя
* деактивация пользователя

Поддерживаются роли:

* admin
* manager

### Аутентификация и авторизация

Реализовано:

* хеширование паролей через bcrypt
* JWT-аутентификация
* middleware для проверки JWT
* ролевая модель доступа (RBAC)

### Аудит действий

Фиксируются действия пользователей:

* изменение статуса посылки
* архивация посылки
* деактивация пользователя

Для каждого действия сохраняются:

* пользователь, выполнивший действие
* старое значение
* новое значение

### Транзакции

Следующие операции выполняются в рамках одной транзакции:

* изменение статуса + история + аудит
* архивация посылки + аудит
* деактивация пользователя + аудит

## Стек технологий

* Go
* PostgreSQL
* sqlx
* JWT
* bcrypt
* Docker

## API

### Авторизация

POST /api/v1/auth/login

### Пользователи

POST /api/v1/users

PATCH /api/v1/users/deactivate?id={id}

### Посылки

POST /api/v1/parcels

GET /api/v1/parcels/track?track_number={track_number}

PATCH /api/v1/parcels/status?id={id}

POST /api/v1/parcels/photos?id={id}

PATCH /api/v1/parcels/archive?id={id}
